▣ ЧАСТИНА 1 — GOOGLE SHEETS
Крок 1 · Створити таблицю і листи
  • Відкрий sheets.google.com → «+» → Порожня таблиця. Назви файл: TIER.MFG
  • Внизу є один лист «Аркуш1» — перейменуй його. Для цього: двічі клікни на назву вкладки → введи ЗАМОВЛЕННЯ
  • Натисни «+» ще чотири рази і назви листи точно так (включно з пробілами і дужками):
    ЗАМОВЛЕННЯ  ·  ВИРОБИ TIER  ·  ДОВІДНИК  ·  АРХІВ
Крок 2 · Лист ЗАМОВЛЕННЯ — структура
A1: № B1: Дата C1: Назва D1: Дедлайн E1: Tier F1: Статус G1: Початок H1: Кінець I1: Вироби
  • Колонка A — автонумерація. Клікни на A2, встав формулу (крапка з комою — бо українська локаль):
    =ЯКЩО(C2="";"";РЯДОК()-1)
    Виділи A2 → потягни за кутик донизу до A1000
  • Колонка D — формат дати. Виділи D2:D1000 → Формат → Число → Дата
  • Колонка G і H — формат дати. Те саме: Формат → Число → Дата
  • Колонка E — мітка Tier. Буде «ТАК» або порожньо — нічого окремо не налаштовуй
  • Колонка F — статус. Виділи F2:F1000 → Дані → Перевірка даних → Додати правило → Розкривний список → впиши через кому:
    Не почато,Розкрій,Цех,Готово,Закрито
  • Колонка I — вироби у форматі JSON. Заповнюється автоматично з HTML — не чіпай вручну
Крок 3 · Лист АРХІВ — структура
  • Той самий рядок заголовків що і в ЗАМОВЛЕННЯ — скопіюй рядок 1 з листа ЗАМОВЛЕННЯ і встав тут
  • Дані сюди потрапляють автоматично з HTML коли замовлення закривається
Крок 4 · Умовне форматування листа ЗАМОВЛЕННЯ (за бажанням)
  • Виділи рядки A2:I1000 → Формат → Умовне форматування → Додати правило
  • В полі «Формула» — увага: крапка з комою, бо українська локаль
  • Зелений (Готово):
    =$F2="Готово"
    Колір фону: #0d2b1a · Текст: #2ecc71
  • Жовтий (Цех):
    =$F2="Цех"
    Фон: #2a2208 · Текст: #f5c518
  • Помаранчевий (Розкрій):
    =$F2="Розкрій"
    Фон: #2a1505 · Текст: #f97316
  • Червоний (протерміновані):
    =І($D2<СЬОГОДНІ();$F2<>"Готово";$F2<>"Закрито")
    Фон: #2b0d0d · Текст: #e74c3c
  • Порядок правил: протерміновані → зверху (вищий пріоритет)
▣ ЧАСТИНА 2 — GOOGLE APPS SCRIPT (API)
Крок 5 · Відкрити редактор Apps Script
  • У Google Sheets відкрий меню Розширення → Apps Script (або Extensions → Apps Script якщо інтерфейс англійський)
  • Відкриється нова вкладка з редактором. Видали весь вміст файлу Код.gs (Ctrl+A → Delete)
  • Скопіюй весь код нижче і встав його (Ctrl+V)
  • Знайди перший рядок const SS_ID = "ВАШ_ID_ТУТА" і заміни ВАШ_ID_ТУТА на ID своєї таблиці
  • ID таблиці — це частина URL між /d/ і /edit:
    https://docs.google.com/spreadsheets/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqpt/edit ↑ це і є ID — копіюй тільки цю частину
  • Натисни 💾 Зберегти (або Ctrl+S)
Крок 6 · Код Apps Script — вставити повністю
// ═══════════════════════════════════════════════ // TIER.MFG — Apps Script API // Вставити повністю в Код.gs, замінивши весь вміст // ═══════════════════════════════════════════════ const SS_ID = "ВАШ_ID_ТУТА"; // ← замінити на ID своєї таблиці const SHEET_ORDERS = "ЗАМОВЛЕННЯ"; const SHEET_ARCHIVE = "АРХІВ"; // ── CORS-заголовки для запитів з браузера ── function corsHeaders() { return { "Access-Control-Allow-Origin": "*", "Access-Control-Allow-Methods": "GET, POST", "Access-Control-Allow-Headers": "Content-Type" }; } function makeResponse(data) { const output = ContentService .createTextOutput(JSON.stringify(data)) .setMimeType(ContentService.MimeType.JSON); return output; } // ── GET: читання даних ── // Перевірка: e може бути undefined якщо запускати вручну в редакторі. // Через браузер (після деплою) — завжди є. function doGet(e) { try { const action = (e && e.parameter) ? e.parameter.action : ""; const ss = SpreadsheetApp.openById(SS_ID); if (action === "getOrders") { return makeResponse(readSheet(ss, SHEET_ORDERS)); } if (action === "getArchive") { return makeResponse(readSheet(ss, SHEET_ARCHIVE)); } // Якщо action невідомий — повернути порожній масив return makeResponse([]); } catch (err) { return makeResponse({ error: err.toString() }); } } // ── POST: запис даних ── function doPost(e) { try { const payload = JSON.parse(e.postData.contents); const ss = SpreadsheetApp.openById(SS_ID); const action = payload.action; if (action === "addOrder") { const sh = ss.getSheetByName(SHEET_ORDERS); const lastNo = getLastOrderNo(sh); sh.appendRow([ lastNo + 1, payload.created, payload.name, payload.deadline, payload.tier ? "ТАК" : "", "Не почато", "", "", payload.productsJson ]); return makeResponse({ ok: true, no: lastNo + 1 }); } if (action === "updateOrder") { const sh = ss.getSheetByName(SHEET_ORDERS); const row = findRowByNo(sh, payload.no); if (!row) return makeResponse({ ok: false, error: "not found" }); // Колонки: A=1(№) B=2(Дата) C=3(Назва) D=4(Дедлайн) E=5(Tier) // F=6(Статус) G=7(Початок) H=8(Кінець) I=9(Вироби) const colMap = { status: 6, workStart: 7, workEnd: 8, productsJson: 9 }; const col = colMap[payload.field]; if (col) sh.getRange(row, col).setValue(payload.value); return makeResponse({ ok: true }); } if (action === "closeOrder") { const sh = ss.getSheetByName(SHEET_ORDERS); const row = findRowByNo(sh, payload.no); if (!row) return makeResponse({ ok: false, error: "not found" }); // Оновити статус на Закрито, потім скопіювати в АРХІВ і видалити sh.getRange(row, 6).setValue("Закрито"); const rowData = sh.getRange(row, 1, 1, 9).getValues()[0]; const archSh = ss.getSheetByName(SHEET_ARCHIVE); archSh.appendRow(rowData); sh.deleteRow(row); return makeResponse({ ok: true }); } return makeResponse({ ok: false, error: "unknown action" }); } catch (err) { return makeResponse({ error: err.toString() }); } } // ── Допоміжні функції ── function readSheet(ss, sheetName) { const sh = ss.getSheetByName(sheetName); if (!sh) return []; const data = sh.getDataRange().getValues(); if (data.length < 2) return []; const headers = data[0]; return data.slice(1) .filter(r => r[0] !== "" && r[0] !== null) .map(r => { const obj = {}; headers.forEach((h, i) => { // Дати перетворюємо в рядок формату YYYY-MM-DD if (r[i] instanceof Date) { obj[h] = Utilities.formatDate(r[i], Session.getScriptTimeZone(), "yyyy-MM-dd"); } else { obj[h] = r[i]; } }); return obj; }); } function getLastOrderNo(sh) { const data = sh.getDataRange().getValues(); if (data.length < 2) return 0; let max = 0; for (let i = 1; i < data.length; i++) { const n = parseInt(data[i][0]); if (!isNaN(n) && n > max) max = n; } return max; } function findRowByNo(sh, no) { const data = sh.getDataRange().getValues(); for (let i = 1; i < data.length; i++) { if (String(data[i][0]) === String(no)) return i + 1; } return null; }
Крок 7 · Опублікувати як веб-застосунок (Deploy)
  • В редакторі Apps Script натисни синю кнопку «Розгорнути» (або Deploy) → «Нове розгортання»
  • У вікні що відкрилось: натисни шестерінку ⚙️ поруч з «Тип» → обери «Веб-застосунок»
  • Заповни поля:
    Опис: TIER.MFG API (або будь-що) Виконувати як: Я (your@gmail.com) Хто має доступ: Усі ← обов'язково, інакше HTML не зможе звертатись
  • Натисни «Розгорнути» → Google попросить авторизацію → натисни «Авторизувати доступ» → вибери свій Google-акаунт → «Додатково» → «Перейти до TIER.MFG (небезпечно)» → «Дозволити»
  • Скопіюй URL який з'явиться — він виглядає так:
    https://script.google.com/macros/s/AKfycbXXXXXXXXXXXXXXXXXXXX/exec
  • ⚠️ Важливо: при будь-якій зміні коду — Розгорнути → Нове розгортання (не «Керувати розгортаннями» і не «Редагувати»). Інакше зміни не застосуються.
▣ ЧАСТИНА 3 — ПІДКЛЮЧЕННЯ HTML І GITHUB PAGES
Крок 8 · Прописати API URL в HTML файлі
  • Відкрий HTML файл у будь-якому текстовому редакторі (VS Code, Notepad++, навіть звичайний Блокнот)
  • Знайди рядок (він близько на початку тегу <script>):
    const API_URL = null;
  • Заміни null на URL з кроку 7 в лапках:
    const API_URL = "https://script.google.com/macros/s/AKfycbXXXX.../exec";
  • Збережи файл (Ctrl+S)
  • Відкрий HTML в браузері (двічі клікни на файл) — при першому завантаженні дані підтягнуться з Google Sheets
Крок 9 · Розмістити на GitHub Pages (безкоштовний хостинг)
  • Якщо немає акаунту — зареєструйся на github.com (безкоштовно)
  • Після входу натисни «+» у правому верхньому куті → «New repository»
  • Заповни:
    Repository name: tier-mfg (або будь-яка назва без пробілів) Visibility: Public ← обов'язково Public для GitHub Pages
  • Натисни «Create repository»
  • На сторінці репозиторію натисни «uploading an existing file» → перетягни свій HTML файл → натисни «Commit changes»
  • Важливо: перейменуй файл на index.html — тоді GitHub Pages відкриватиме його за замовчуванням
  • Тепер увімкни GitHub Pages: Settings → Pages → Source → Deploy from a branch → main → / (root) → Save
  • Через ~1 хвилину твій сайт буде доступний за адресою:
    https://твій_нікнейм.github.io/tier-mfg/
  • Щоб оновити файл на GitHub: відкрий репозиторій → клікни на index.html → іконка олівця ✏️ → встав новий вміст → «Commit changes»
Крок 10 · Перевірка що все працює
  • Відкрий сайт на GitHub Pages → вкладка «Замовлення» має завантажити дані з Google Sheets (якщо вони є)
  • Створи тестове замовлення → відкрий Google Sheets → переконайся що новий рядок з'явився в листі ЗАМОВЛЕННЯ
  • Зміни статус в HTML → перевір що статус змінився в колонці F таблиці
  • Якщо дані не підтягуються — відкрий консоль браузера (F12 → Console) і подивись на помилки. Найчастіша: CORS або неправильний URL
  • Якщо бачиш помилку CORS — переконайся що в Apps Script «Хто має доступ» = Усі, і зроби нове розгортання
⚠️ Типові помилки і рішення
  • TypeError: Cannot read properties of undefined (reading 'parameter') — ця помилка виникає коли запускаєш doGet вручну в редакторі Apps Script. Це нормально — функція очікує запит з браузера. Через URL після деплою все працює
  • Дані не зберігаються після перезавантаження — означає що API_URL = null. Перевір що замінив на правильний URL
  • Формули у Sheets не працюють — перевір що використовуєш крапку з комою (;) як роздільник, а не кому. В українській локалі Google Sheets роздільник — крапка з комою
  • Після зміни коду Apps Script нічого не змінилось — зроби «Нове розгортання» (не редагуй існуюче). URL при цьому зміниться — оновіть API_URL в HTML
  • GitHub Pages показує 404 — переконайся що файл називається index.html (не TIER_MFG_final.html), і що Pages увімкнено в Settings
